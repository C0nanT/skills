# `/triage` — Triagem de Issues

## O que é

Uma skill que move issues do issue tracker através de um state machine de papéis de triagem. Permite gerenciar o fluxo de bugs e feature requests de forma estruturada.

## Para que serve

- Ver quais issues precisam de atenção agora
- Avaliar um bug ou feature request específico
- Preparar issues para serem executadas por um agente AFK
- Gerenciar o workflow de issues sem precisar entrar no GitHub/GitLab manualmente

## Como invocar

Em linguagem natural depois de `/triage`:

```
/triage
Show me anything that needs my attention

/triage
Let's look at #42

/triage
Move #42 to ready-for-agent

/triage
What's ready for agents to pick up?
```

**Pré-requisito**: rode `/setup-skills` antes de usar pela primeira vez.

## Como funciona

### State machine

Cada issue triada tem exatamente um papel de **categoria** e um papel de **estado**:

**Categorias:**
- `bug` — algo está quebrado
- `enhancement` — nova feature ou melhoria

**Estados:**
- `needs-triage` — maintainer precisa avaliar
- `needs-info` — aguardando mais informação do reporter
- `ready-for-agent` — completamente especificado, pronto para agente AFK
- `ready-for-human` — precisa de implementação humana
- `wontfix` — não será acionado

Transições normais: sem label → `needs-triage` → (`needs-info` | `ready-for-agent` | `ready-for-human` | `wontfix`). `needs-info` volta para `needs-triage` quando o reporter responde.

### Disclaimer obrigatório

Todo comentário ou issue postado durante a triagem começa com:
```
> *This was generated by AI during triage.*
```

### Ver o que precisa de atenção

Apresenta três buckets, do mais antigo ao mais novo:
1. **Sem label** — nunca triado
2. **`needs-triage`** — avaliação em progresso
3. **`needs-info` com atividade recente do reporter** — precisa de reavaliação

### Triagem de uma issue específica

1. **Coleta contexto** — lê o corpo completo, comentários, labels, reporter. Lê notas de triagem anteriores para não repetir perguntas já respondidas.

2. **Recomenda** — diz categoria e estado recomendados com justificativa, mais um resumo da codebase relevante. Espera sua direção.

3. **Reproduz** (só bugs) — antes de qualquer grilling, tenta reproduzir: lê os passos do reporter, rastreia o código relevante, roda testes. Reporta o resultado.

4. **Grilla se necessário** — se a issue precisa de mais detalhes, entra numa sessão `/grill-with-docs`.

5. **Aplica o resultado:**
   - `ready-for-agent` → posta um agent brief (instruções completas para um agente executar sem contexto humano)
   - `ready-for-human` → mesmo formato do agent brief, mas nota por que não pode ser delegado
   - `needs-info` → posta template de notas de triagem com perguntas específicas e acionáveis
   - `wontfix` (bug) → explicação educada, fecha a issue
   - `wontfix` (enhancement) → escreve em `.out-of-scope/`, linka do comentário, fecha

### Override rápido de estado

Se você diz "mova #42 para ready-for-agent", a skill confia em você e aplica diretamente. Confirma o que vai fazer, depois age. Sem grilling.

## Exemplo de uso

```
/triage
Show me what needs attention
```

O agente lista: "3 issues sem label, 1 em needs-triage, 2 em needs-info com resposta do reporter." Você escolhe qual atacar.

```
/triage
Let's look at #15
```

O agente lê a issue #15 (suposto bug de login), tenta reproduzir, recomenda `bug / ready-for-agent` com justificativa, você confirma, ele posta o agent brief.
